## 1. 剧本结构概览

一个剧本由以下模块组成：

| 模块 | 说明 | 是否必填 |
|------|------|----------|
| 基本信息 | 剧本名称、描述 | 是 |
| 写作规则 | 文风、禁止事项、写作要求 | 否 |
| 数值维度 | 驱动剧情的数值属性 | 推荐 |
| 角色设定 | 剧中角色定义 | 推荐 |
| 关键事件 | 满足条件时触发的特殊事件 | 否 |
| 剧情阶段 | 剧情推进的阶段划分 | 推荐 |
| 结局设定 | 不同条件下的结局 | 否 |
| 开局选项 | 玩家开始前的自定义选择 | 推荐 |

---

## 2. 基本信息

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | 字符串 | 剧本名称，显示在首页 |
| `description` | 字符串 | 剧本简介，帮助玩家了解剧情方向 |

---

## 3. 写作规则

控制 LLM 的写作风格和行为约束。

### 3.1 写作风格 (`rules.writing_style`)

一段描述文风的文本，LLM 会遵循此风格写作。

```
文风像豆瓣/网络高热恋爱文。生活真实语气。直白描写，禁止文艺比喻过多。
```

### 3.2 禁止事项 (`rules.forbidden`)

字符串数组，每条都是 LLM 必须遵守的禁令。

```json
[
  "禁止开局直接上床",
  "禁止见面三天爱得死去活来",
  "所有关系必须循序渐进"
]
```

### 3.3 写作要求 (`rules.requirements`)

字符串数组，描述期望的写作行为。

```json
[
  "长描写、沉浸式互动",
  "每轮必须给选择（自由输入或选项）",
  "情感必须有铺垫"
]
```

## 4. 数值维度

驱动剧情变化的数值属性。每个维度有以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | 字符串 | 唯一标识符，用于事件触发和宏引用 |
| `name` | 字符串 | 显示名称，LLM 通过名称识别和更新数值 |
| `range` | [min, max] | 数值范围 |
| `initial` | [min, max] | 初始值范围（开局时随机取值） |

### 示例

```json
{
  "id": "love",
  "name": "爱意值",
  "range": [0, 100],
  "initial": [85, 95]
}
```

### 设计建议

- **id 用英文**，name 用中文。LLM 通过 name 匹配数值，id 用于程序内部引用
- **range** 建议设为 [0, 100]，便于 LLM 理解
- **initial** 的 min 和 max 可以不同，开局时在范围内随机
- 维度数量建议 5-15 个，太多会分散 LLM 注意力

## 5. 角色设定

定义剧本中的角色。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | 字符串 | 唯一标识符 |
| `name` | 字符串 | 角色名，LLM 用此名称呼角色 |
| `description` | 字符串 | 角色描述，包含性格、身份、与玩家的关系 |

### 示例

```json
{
  "id": "partner",
  "name": "现任",
  "description": "玩家当前的伴侣，温柔但敏感，容易察觉异常"
}
```

### 内置角色 ID

以下 ID 有特殊用途：

| ID | 说明 | 宏引用 |
|----|------|--------|
| `player` | 玩家角色 | `{{player}}` |
| `partner` | 现任/主要NPC | `{{partner}}` |
| `target` | 危险对象/次要NPC | `{{target}}` |

使用内置 ID 时，宏系统可以自动引用角色名。

## 6. 关键事件

满足特定条件时触发的事件，会影响剧情走向。

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | 字符串 | 事件名称 |
| `stage` | 数字 | 限制在哪个阶段触发（0-based），不填则任何阶段 |
| `trigger` | 对象 | 触发条件（详见[条件组合](#11-条件组合)） |
| `description` | 字符串 | 事件描述，帮助 LLM 理解事件内容 |
| `effects` | 对象 | 持续效果（详见[持续效果](#12-持续效果)） |

### 示例

```json
{
  "name": "现任偷看手机",
  "stage": 1,
  "trigger": { "suspicion": { "min": 50, "max": 70, "probability": 0.5 } },
  "description": "现任趁玩家不注意偷看了手机",
  "effects": {
    "sticky": { "dims": { "suspicion": 3 }, "duration": 3 },
    "cooldown": 5
  }
}
```

## 7. 剧情阶段

剧情推进的阶段划分。当满足过渡条件时自动进入下一阶段。

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | 字符串 | 阶段名称 |
| `transition` | 对象 | 进入此阶段的条件（同事件触发条件格式） |

### 示例

```json
[
  { "name": "稳定生活", "transition": { "suspicion": { "min": 20 } } },
  { "name": "危险接触", "transition": { "heartbeat": { "min": 40 } } },
  { "name": "秘密关系" },
  { "name": "结局" }
]
```

- 第一个阶段不需要 transition（开局即进入）
- 最后一个阶段通常不需要 transition（结局由结局设定控制）

---

## 8. 结局设定

不同条件下的结局。

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | 字符串 | 结局名称 |
| `condition` | 对象 | 达成条件（同事件触发条件格式） |
| `description` | 字符串 | 结局描述 |

### 示例

```json
{
  "name": "HE-主动坦白",
  "condition": { "guilt": { "min": 70 }, "love": { "min": 60 } },
  "description": "你鼓起勇气向现任坦白了一切..."
}
```

---

## 9. 开局选项

玩家开始游戏前的自定义选择。

| 字段 | 类型 | 说明 |
|------|------|------|
| `step` | 字符串 | 步骤名称（如"你的性别"） |
| `options` | 数组 | 选项列表，每项有 `label`（显示文本）和 `value`（实际值） |
| `allow_custom` | 布尔 | 是否允许玩家自定义输入 |

### 示例

```json
{
  "step": "你的职业",
  "options": [
    { "label": "普通上班族", "value": "普通上班族" },
    { "label": "设计师", "value": "设计师" },
    { "label": "律师", "value": "律师" }
  ],
  "allow_custom": true
}
```

## 10. 宏系统

在写作规则、禁止事项、角色描述中可以使用宏，运行时自动替换为实际值。

### 可用宏

| 宏 | 替换为 | 示例输出 |
|----|--------|----------|
| `{{player}}` | 玩家角色名 | "玩家" |
| `{{partner}}` | 现任角色名 | "小雨" |
| `{{target}}` | 危险对象角色名 | "上司" |
| `{{stage}}` | 当前阶段名 | "秘密关系" |
| `{{stage_num}}` | 当前阶段编号 | "3" |
| `{{dim:爱意值}}` | 指定维度的当前值 | "85" |
| `{{random:A,B,C}}` | 从列表中随机选一个 | "B" |

### 使用示例

在写作规则中：
```
当前阶段：{{stage}}。{{partner}}正在观察你的行为。
```

在禁止事项中：
```
禁止{{partner}}突然变得不合理
```

在角色描述中：
```
{{target}}是一个充满魅力的人，当前与{{player}}的关系正在升温。
```

## 11. 条件组合

事件触发、阶段过渡、结局达成都使用条件系统。

### 简单条件（单维度）

```json
{ "suspicion": { "min": 50 } }
```

表示：怀疑值 >= 50

### 条件字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `min` | 数字 | 最小值（含） |
| `max` | 数字 | 最大值（含） |
| `probability` | 数字 | 触发概率（0-1），每次检查时独立随机 |

### 组合条件（AND/OR）

当需要多个维度同时满足或任一满足时，使用组合格式：

**全部满足 (AND)**：
```json
{
  "op": "and",
  "conditions": [
    { "dim": "suspicion", "min": 50 },
    { "dim": "jealousy", "min": 30 }
  ]
}
```

**任一满足 (OR)**：
```json
{
  "op": "or",
  "conditions": [
    { "dim": "guilt", "min": 70 },
    { "dim": "stress", "min": 80 }
  ]
}
```

### 向后兼容

旧格式（直接写维度 ID）仍然支持：
```json
{ "suspicion": { "min": 50 }, "jealousy": { "min": 30 } }
```
等价于 AND 逻辑。

## 12. 持续效果

事件触发后可以产生持续影响，增强剧情连贯性。

### 效果类型

| 类型 | 说明 | 字段 |
|------|------|------|
| `sticky` | 指定维度每轮自动加减，持续 N 轮 | `dims`（维度变化量）、`duration`（持续轮数） |
| `cooldown` | 事件触发后 N 轮内不再触发 | 直接写轮数 |

### 示例

```json
{
  "name": "现任偷看手机",
  "effects": {
    "sticky": { "dims": { "suspicion": 3 }, "duration": 3 },
    "cooldown": 5
  }
}
```

**效果说明**：
- 事件触发后，怀疑值每轮自动 +3，持续 3 轮（模拟"怀疑情绪持续发酵"）
- 同一事件 5 轮内不会再次触发（避免重复剧情）

### 使用建议

- **sticky** 适合模拟情绪余波：吵架后怒气慢慢消退、误会后信任慢慢恢复
- **cooldown** 适合避免重复：同一个事件不要连续触发
- `duration` 和 `cooldown` 设为 0 或不填表示不启用

```json
{
  "name": "危险关系模拟器",
  "description": "情感拉扯、谎言压力、道德挣扎。",
  "rules": {
    "writing_style": "文风像豆瓣恋爱文。生活真实语气。直白描写。",
    "forbidden": [
      "禁止开局直接上床",
      "禁止见面三天爱得死去活来",
      "所有关系必须循序渐进"
    ],
    "requirements": [
      "长描写、沉浸式互动",
      "每轮必须给选择"
    ]
  },
  "dimensions": [
    { "id": "love", "name": "爱意值", "range": [0, 100], "initial": [85, 95] },
    { "id": "trust", "name": "信任值", "range": [0, 100], "initial": [85, 95] },
    { "id": "suspicion", "name": "怀疑值", "range": [0, 100], "initial": [0, 10] },
    { "id": "heartbeat", "name": "心动值", "range": [0, 100], "initial": [0, 0] }
  ],
  "characters": [
    { "id": "player", "name": "玩家", "description": "由玩家自定义" },
    { "id": "partner", "name": "现任", "description": "玩家当前的伴侣" },
    { "id": "target", "name": "危险对象", "description": "新的情感诱惑" }
  ],
  "events": [
    {
      "name": "现任试探",
      "stage": 1,
      "trigger": { "suspicion": { "min": 30, "max": 50, "probability": 0.3 } },
      "description": "现任开始旁敲侧击"
    },
    {
      "name": "修罗场",
      "stage": 3,
      "trigger": {
        "op": "and",
        "conditions": [
          { "dim": "suspicion", "min": 50 },
          { "dim": "heartbeat", "min": 60 }
        ]
      },
      "description": "现任和危险对象在同一场合出现",
      "effects": {
        "sticky": { "dims": { "suspicion": 5 }, "duration": 2 },
        "cooldown": 3
      }
    }
  ],
  "stages": [
    { "name": "稳定生活", "transition": { "suspicion": { "min": 20 } } },
    { "name": "危险接触", "transition": { "heartbeat": { "min": 40 } } },
    { "name": "秘密关系" },
    { "name": "结局" }
  ],
  "endings": [
    {
      "name": "HE-主动坦白",
      "condition": { "love": { "min": 60 } },
      "description": "你鼓起勇气向现任坦白了一切..."
    }
  ],
  "setup": [
    {
      "step": "你的性别",
      "options": [
        { "label": "男", "value": "男" },
        { "label": "女", "value": "女" }
      ]
    }
  ]
}
```
