# 剧本模拟器优化任务列表

本文档按优先级整理当前项目的缺陷修复与优化任务，方便后续拆分开发。

## P0：稳定性与安全底座

1. **改造 LLM 输出协议为结构化 JSON**
   - 问题：当前依赖正则从自然语言中提取选项、数值更新、关键事件，模型稍微改格式就会失败。
   - 目标：要求模型输出固定 JSON block，包含 `narrative`、`options`、`values`、`keyEvent`、`stageHint`。
   - 验收：非法 JSON 能提示重试；数值字段只接受已定义维度 ID。

2. **为导入剧本增加 schema 校验**
   - 问题：导入 JSON 只做 `JSON.parse`，没有检查维度、角色、事件、阶段引用是否合法。
   - 目标：保存和导入前运行统一校验，返回错误与警告。
   - 验收：重复 ID、事件阶段越界、条件引用不存在维度、缺少 player 角色都能被拦截。

3. **处理 API Key 安全风险**
   - 问题：API Key 存在浏览器 IndexedDB，并由前端直接请求模型服务。
   - 目标：设计可选后端代理或 serverless token broker。
   - 验收：生产模式下前端不持久化长期 API Key。

4. **为会话状态增加快照或重放机制**
   - 问题：重新生成 AI 回复时只删除消息，数值、阶段、事件效果、记忆不会自动回滚。
   - 目标：每轮保存状态快照，或从消息历史重放状态。
   - 验收：重生成任意一轮后，状态与新历史一致。

5. **修复自动记忆异步保存问题**
   - 问题：自动总结是异步的，但消息写入后没有等待总结完成，可能保存旧的 `memoryState`。
   - 目标：增加 pending summary flush，或让消息写入方法返回可等待的 Promise。
   - 验收：达到总结间隔后刷新页面，最新记忆仍然存在。

## P1：核心玩法一致性

1. **数值初始化支持区间随机与 seed**
   - 当前只取初始区间下限。需要明确支持固定值或区间随机，并保存 seed 以便复现。

2. **数值更新按维度范围 clamp**
   - LLM 输出超出范围时，自动限制在 `range` 内，并在调试日志中记录。

3. **事件触发增加 once、cooldown、maxTriggers**
   - 避免同一事件在满足条件时反复触发。

4. **阶段跳转支持连续推进**
   - 当一次数值变化跨过多个阶段条件时，可以连续检查后续阶段。

5. **结局判定接入主流程**
   - 当前结局配置存在，但聊天流程中没有明显的结束状态与结局渲染。

## P2：工程化

1. **补齐 `package.json` 与 lockfile**
   - 固定 esbuild、lint、test、build 命令，避免依赖 `npx` 临时下载。

2. **为纯函数增加单元测试**
   - 覆盖条件判断、事件触发、数值提取、宏替换、会话分支。

3. **拆分 prompt-builder**
   - 拆成宏系统、上下文裁剪、输出协议、模型适配四部分。

4. **统一 LLM adapter**
   - 处理 OpenAI-compatible、DeepSeek thinking、流式/非流式、timeout、abort、重试。

5. **PWA 资源与缓存策略修复**
   - 补齐 icon 文件，构建时生成缓存清单，API 请求使用 network-only。

## P3：编辑器与用户体验

1. **编辑器增加校验面板**
   - 保存前展示错误和警告，不让用户只在运行时发现剧本坏了。

2. **导入失败显示定位信息**
   - JSON 解析错误、schema 错误、引用错误分层展示。

3. **首页卡片信息密度优化**
   - 更清晰展示剧本数量、会话数量、最近更新时间和主要操作。

4. **聊天页增加状态变化时间线**
   - 让玩家能回看每轮数值变化、触发事件、阶段推进。

5. **模型配置增加预设**
   - 保存常见供应商 endpoint、模型名、thinking 参数组合。
